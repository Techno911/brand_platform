# Gotchas: other (не базовая индустрия)

## Общие правила
- Любой вывод должен иметь цитату-источник из собранных данных.
- Methodology-canon имеет приоритет над индустриальной интуицией маркетолога.
- Если нет готового gotcha-файла, маркетолог **обязательно** оставляет в project.industry_notes свою заметку для Чиркова: "типичный конкурент", "регуляторика", "жизненный цикл клиента".

## Что всегда проверять
1. В месседже нет цифр и денежных маркеров.
2. Миссия без слов "лучший/№1/доминировать".
3. Архетип из 12 канонических.
4. Позиционирование с WE-парой, включая `we_are_not`.
5. Легенда с датами, без маркетинговых прилагательных.

## Что допускается с оговоркой
- Если индустрия **B2B2C** (школы, страхование), то часть триггеров заимствуется из B2C-психологии, но сегмент "покупателя" и "пользователя" разделяется явно.
- Если индустрия **услуги фрилансеров** (дизайн, фотография) — месседж строится вокруг конкретного формата работы, а не абстрактного "творчества".

## Чек-лист выхода
- [ ] Маркетолог в Чиркова докладывает конкретные gotcha, собранные за проект.
- [ ] Чирков принимает решение — добавлять ли эту индустрию в knowledge base как новый файл.
