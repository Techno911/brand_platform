# Industry context: other (не из 4 базовых индустрий MVP)

Этот файл загружается, когда проект отмечен как `other` — не стоматология, мебель, ресторан или салон.

## Чем это отличается от 4 базовых
- **Нет готового паспорта сегментов** — маркетолог обязан собрать его на Стадии 1 вручную.
- **Нет готовых gotcha-правил по отрасли** — полагается только на methodology-canon.
- **Контекстная маржа LLM-вызовов выше** — каждый вопрос требует больше контекста от маркетолога.

## Что усилить в workflow
1. На Стадии 1 **удвоить количество отзывов** — 40+ вместо 20+.
2. На Стадии 2 собственник должен **сам сформулировать** 3 примерных миссии, далее Claude улучшает (а не генерирует с нуля).
3. На Стадии 3 архетип выбирается **только с evidence**. Без ≥3 цитат из сессии — не утверждается.
4. На Стадии 4 все 4 теста прогоняются **с живыми респондентами**, не синтетически.

## Что запрещено
- Использовать позиционирование/архетипы из других индустрий (стоматология и т.п.) как шаблон.
- Обещать что-то на основе общих маркетинговых best practices — без источников из конкретной индустрии клиента.

## Чек-лист
- [ ] Индустрия клиента явно описана в brief.
- [ ] Собраны ≥40 отзывов из минимум 3 источников.
- [ ] Собраны ≥5 прямых + ≥3 косвенных конкурента.
- [ ] methodology-compliance-check проходит строго.
- [ ] Маркетолог явно подтверждает, что отсутствие готовых gotcha — ответственность клиента за дополнительный сбор данных.
